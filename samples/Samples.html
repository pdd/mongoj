 PortletWithService
 This is all-in-one war.
It contains the service and portlet in one war.
Easiest to try by just dropping the war into a running Liferay instance.
  1. Download war
  2. Start mongoDB with all defaults
  3. Deploy portlet to Liferay
  4. Add to page and test
2
 PortletAndService
 This is a more realistic example where the service and portlet are bundled as different wars.
Service is deployed with service interfaces (service.jar) in global classpath while the service implementation (service-impl.jar) is in service war.
This makes it possible to consume the service from any other webapp deployed on to the same container.
The portlet consumes the service and provides UI for the service.
